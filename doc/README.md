# NeuG Documentation

This directory contains the source of documentation for NeuG, including both user guides and API references.

## Quick Start

### 0. Requirements

- Python 3.7+
- Doxygen (for C++ API documentation)
- NeuG packages (for API introspection)

### 1. Install Python Dependencies

```bash
make dependencies
```

### 2. Build Documentation

#### Full Build (Recommended)
```bash
make html
```
This command will:
- Generate Doxygen XML for C++ API
- Generate Python and C++ API documentation 
- Build Sphinx documentation

#### Incremental Build
You can do each of the above steps separately, in case that certain
step have been completed before.
```bash
# Only generate Doxygen XML
make doxygen

# Only regenerate API docs (without full Sphinx rebuild)
make api-docs

# Only rebuild Sphinx (after API docs are generated)
make html-only
```

### 3. View Documentation

Start a local web server:
```bash
python3 -m http.server 8080 -d build/html
```

Then open `http://localhost:8080` in your browser.

## API Documentation

The API documentation is automatically generated and optimized for Nextra:

### Python API
- **Tool**: `pydoc-markdown` for pure Markdown generation
- **Source**: Python docstrings from `/tools/python_bind`
- **Output**: Individual module files (`database.md`, `connection.md`, etc.)
- **Features**: 
  - Item list formatting for parameters/returns
  - Automatic code detection
  - Clean module structure

### C++ API
- **Tool**: Custom script processing Doxygen XML
- **Source**: C++ header files with Doxygen comments  
- **Output**: Namespace files (`gs.md`, `server.md`, etc.)
- **Features**:
  - Smart function signature formatting for long parameter lists
  - Key class prioritization (⭐ indicators)
  - Filtered to exclude protobuf-generated namespaces

### Generated Files
- **`index.md`**: Rich API overview with examples and quick start
- **`index.rst`**: Simple toctree for Sphinx compatibility
- **`_meta.ts`**: Nextra navigation metadata for website integration
- **Individual documentation files**: Per-module/namespace content

Both scripts now automatically clean previously generated files before creating new ones:

- Preserves manually created files (like `index.md`)
- Provides `--clean-only` and `--no-clean` options for flexibility

#### Nextra Integration
All generated documentation includes:
- `_meta.ts` files for automatic navigation
- Proper Markdown formatting for Nextra compatibility
- Optimized file structure for website deployment

### Clean Build
```bash
make clean
```

### Configuration

#### Python API Configuration
Edit `source/_scripts/pydoc-markdown.yml` to modify:
- Module list and search paths
- Processor filters and transformations
- Output formatting options

#### C++ API Configuration  
Edit `source/_scripts/cpp_key_classes.json` to modify:
- `target_namespaces`: List of C++ namespaces to include in documentation
- `key_classes`: Classes that receive detailed documentation with ⭐ indicators
- `priority_display` and `full_documentation`: Feature flags

Edit `source/_scripts/generate_cpp_docs.py` to modify:
- Function signature formatting thresholds (line 517)
- Documentation generation logic

## Following-ups: Integration with Nextra

### Overview

The NeuG project uses [Nextra](https://nextra.site/) to build the main documentation website. Nextra is a Next.js-based static site generator that provides:

- Modern, responsive documentation UI
- Multi-language support with automatic translation
- Advanced search functionality
- Interactive components and live examples

### Architecture

The documentation workflow follows this architecture:

```
doc/source/           →    website/content/    →    Deployed Website
(Sphinx source)            (Nextra source)          (nextra.site)
     │                           │                       │
     │ qwen-translator sync      │ next build           │
     └─────────────────────────→ └─────────────────────→ 
```

### Synchronization with qwen-translator

We use `qwen-translator`, an npm-based tool, to bridge Sphinx and Nextra:

#### Installation
```bash
npm install -g qwen-translator
```

#### Sync Documentation
```bash
# Clone NeuG's website repo
git clone https://github.com/graphscope/neug-web
cd neug-web
# Sync documentation from doc/source to website/content
qwen-translator sync
```

#### Features

**qwen-translator** provides:

1. **Content Synchronization**: Automatically copies and converts Markdown files from `doc/source/` to `website/content/`

2. **Multi-language Translation**: Leverages the Qwen3 AI model to automatically translate documentation:
   - English (source) → Chinese
   - Maintains technical accuracy and context
   - Preserves code blocks and formatting

3. **Metadata Generation**: Creates necessary `_meta.ts` files for Nextra navigation

#### Workflow

1. **Generate documentation** in `doc/source/` using Sphinx
2. **Sync to Nextra** using `qwen-translator sync`
3. **Build website** using Next.js in the `website/` directory
4. **Deploy** the final website

#### Configuration

The translation behavior is configured in `website/translation.config.json`:
- Translation rules and patterns
- File inclusion/exclusion filters  
- Language-specific customizations
- AI model parameters for Qwen3

### Benefits

This integrated approach provides:

- **Single Source of Truth**: All content originates from Sphinx
- **Automatic Translation**: AI-powered multi-language support
- **Modern UI**: Nextra's advanced documentation features
- **Developer Friendly**: Familiar Markdown + React workflow
- **SEO Optimized**: Static site generation for better performance
